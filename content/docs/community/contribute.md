## Getting Started

### 1. Find an Issue

Browse open issues on GitHub:
- **Good First Issue**: Perfect for newcomers
- **Help Wanted**: Community assistance needed
- **Bug**: Report or fix bugs
- **Enhancement**: New feature requests

### 2. Fork the Repository

```bash
# Fork on GitHub, then clone
git clone https://github.com/YOUR_USERNAME/hedgecore.git
cd hedgecore
```

### 3. Create a Branch

```bash
git checkout -b feature/your-feature-name
```

### 4. Make Changes

- Follow coding standards
- Add tests for new features
- Update documentation
- Test thoroughly

### 5. Commit Changes

```bash
git add .
git commit -m "Add: brief description of changes"
```

Use conventional commits:
- `Add:` New feature
- `Fix:` Bug fix
- `Update:` Improve existing feature
- `Docs:` Documentation changes

### 6. Push and Create PR

```bash
git push origin feature/your-feature-name
```

Create Pull Request on GitHub with:
- Clear description
- Related issue numbers
- Testing performed
- Screenshots (if UI changes)

## Contribution Guidelines

### Code Quality

- Follow existing code style
- Write clear, commented code
- Add unit tests for new features
- Ensure all tests pass

### Smart Contract Standards

- Use OpenZeppelin libraries
- Follow Solidity best practices
- Add NatSpec documentation
- Consider gas optimization

### Frontend Standards

- TypeScript for type safety
- Responsive mobile design
- Accessibility compliance (WCAG)
- Component documentation

### Documentation Standards

- Clear and concise writing
- Code examples where relevant
- Proper markdown formatting
- Check for typos and grammar

## Review Process

1. **Submission**: Submit PR with description
2. **Review**: Team reviews code and provides feedback
3. **Revisions**: Address review comments
4. **Approval**: PR approved by maintainers
5. **Merge**: Changes merged to main branch

## Bug Bounty Program

### Scope

Security vulnerabilities in:
- Smart contracts
- Frontend application
- Infrastructure

### Rewards

Based on severity:
- **Critical**: $5,000 - $10,000
- **High**: $2,000 - $5,000
- **Medium**: $500 - $2,000
- **Low**: $100 - $500

### Reporting

**Email**: security@hedgecore.io

Include:
- Vulnerability description
- Steps to reproduce
- Potential impact
- Suggested fix (optional)

### Rules

- Responsible disclosure required
- No public disclosure before fix
- No exploitation of vulnerabilities
- Follow coordinated disclosure timeline
